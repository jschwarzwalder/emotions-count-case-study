# Emotions Count — UX / Product Support Case Study

## Overview
Emotions Count was a classroom web application designed to allow students to quickly submit emotional check-ins and flag concerns for teacher follow-up.

The system included a teacher dashboard for monitoring student responses and identifying risk indicators.

---

## My Role
Lead Developer / UX Designer / Product Builder

---

## Problem
Teachers needed a fast, low-friction way to:
- Understand student emotional states
- Identify students needing follow-up
- View class-wide emotional trends

---

## Solution
I designed and built a full-stack web application using:
- PHP
- MySQL
- Bootstrap
- jQuery

Key features:
- Student emotional check-in system
- Teacher dashboard with color-coded risk indicators
- Automated highlighting of concerning responses
- Parent email notification system
- Data deletion tools for privacy compliance

---

## UX / QA Approach
- Built early paper prototypes with stakeholders
- Iterated based on classroom feedback
- Conducted usability testing in real classroom environments
- Adjusted workflows based on teacher observation and feedback
- Improved clarity of dashboard based on real usage patterns

---

## Impact
- Improved teacher ability to monitor student emotional trends
- Enabled faster identification of at-risk students
- Expanded system usage beyond initial classroom
- Incorporated feedback from multiple educators into iterative releases

---

## Key Learnings
- Importance of low-friction UX in classroom environments
- Value of iterative feedback from real users
- Balancing privacy requirements with usability
- Designing dashboards for quick decision-making
